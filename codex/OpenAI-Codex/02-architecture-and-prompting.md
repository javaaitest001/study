# 02. Codex의 동작 원리와 아키텍처

## 1. 학습 데이터 및 성능 트레이닝
* Codex는 수십 억 줄의 공개 소스코드가 포함된 대규모 데이터셋(GitHub Public Repositories)을 학습했습니다.
* 일반 GPT-3 모델이 자연어 문맥을 파악하는 데 특화되어 있다면, Codex는 프로그래밍 언어의 구문(Syntax), 라이브러리 활용법, 함수 간의 인과관계를 추론하는 능력이 극대화되어 있습니다.

## 2. 프롬프트 엔지니어링 (Prompt Engineering)
Codex의 성능을 끌어올리기 위해서는 '컨텍스트(Context)'를 어떻게 주느냐가 핵심입니다.

### 💡 효과적인 코드 생성 프롬프트 구조
1. **역할 정의 및 목표 설정:** 주석문 형식으로 작성할 기능의 목적을 명확히 명시합니다.
2. **제한 사항 제시:** 사용할 라이브러리나 반환값의 타입, 예외 처리 여부를 미리 가이드합니다.
3. **Context 제공:** 기존에 작성된 클래스 구조나 변수명을 프롬프트 윗부분에 포함하면, Codex가 이를 인지하고 일관성 있는 코드를 생성합니다.

> **Note:** Codex는 단순한 문장 완성(Completion) 모델에 가깝기 때문에, 주석의 마지막 단어나 함수 선언부의 시작 부분을 자연스럽게 이어 붙여주는 방식으로 유도할 때 가장 좋은 결과가 나옵니다.
